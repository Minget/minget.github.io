用GitHub Pages搭建一个免费的个人博客网站

Begin！

安装软件

必要软件：Node.js、Git

  • 要用到Node.js的NPM包管理器

  • Git安装时勾选Add a Git Bash Profle t Windows Terminal

可选软件：Clash、Typora、VsCode、

  • 用Clash代理登录GitHub

  • Typora图像设置：插入图片时复制到./${filename}

  • VsCode安装插件：Open in External App，此插件支持在VsCode中右键打开.md文档，Ctrl+Shift+P打开用户设置settings.json添加

网络问题

NPM镜像源：设置为淘宝镜像源

  • 设置为淘宝镜像源

  • 查询当前的镜像源

  • 设置回默认镜像源

远程仓库镜像源：设置为SSH连接

  1. 创建SSH Key，输入命令后多按几次空格设置为空密码

  2. 用Everything搜索.ssh，找到id_rsa.pub文件，复制里面的内容

  3. 登录GitHub⇨Settings⇨把复制的内容填入SSH and GPG Keys

  4. 验证是否成功

  5. 打开项目的Code窗口，切换为SSH，此时项目的地址为：git@github.com:<username>/<username>.github.io.git

配置网站

远程仓库：GitHub Pages

  1. 注册并登录GitHub
  2. 创建一个名为<username>.github.io的仓库
  3. 进入仓库的Settings⇨Pages，可以看到Your site is live at https://<username>.github.io/

博客框架：Hexo

  1. 全局安装Hexo

  2. 建立本地博客仓库

  3. 本地预览

博客主题：Stellar

  1. 安装Stellar

  2. _config.yml中启用Stellar

  3. 自定义站点配置文件：_config.yml的site部分

  4. 博客根目录下新建_config.stellar.yml，此为用户自定义的主题配置文件

网站插件

直接安装

  • 同步：hexo-browsersync

需改配置

  • 布署：hexo-deployer-git

  • 链接：hexo-abbrlink2

  • 图片：hexo-asset-image

  • 分类：hexo-auto-category

发布文章

基本了解

  • 文章类型：
    • Post：完成的文章，默认位于./source/_posts
    • Draft：未完成的文章，默认位于./source/_drafts
  • 模板类型：
    • Post：文章模板，默认位于./scaffold/post.md
    • Draft：草稿模板，默认位于./scaffold/draft.md
  • 页面类型：
    • Index：首页，Stellar主题默认显示最近文章
    • Posts：文章页，包含完整文章内容的页面
    • Categories：分类页，显示已创建的所有分类
    • Tags：标签页，显示已创建的所有标签
    • Archives：归档页，根据时间列出所有文章

基本操作

  1. 修改文章页模板：./scaffold/post.md

  2. 生成带有分类的文章：hexo n post -p <categoriesname>/<postname>

  3. 用Typora编辑文章

  4. 布署到远程服务器：hexo clean&&hexo g&&hexo d -m "<message>"

  5. 至少布署过一次后，进入.deploy_git隐藏文件夹

备注

  • HTTPS和SSH的区别

    • HTTPS可以随意克隆GitHub上任何人的项目，而SSH要求本地计算机必须拥有对应GitHub账户的SSH Key

    • HTTPS在push时需要用户名和密码，而SSH经过上述设置不需要密码
